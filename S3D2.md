### 1. Entradas y Salidas
* **Entrada (Input):** Es la forma en que el programa recibe información del mundo exterior, generalmente del usuario a través del teclado. (Ejemplo: `input("Ingrese su nombre: ")`).
* **Salida (Output):** Es cómo el programa muestra los resultados al usuario. (Ejemplo: `print("Hola Mundo")`).

### 2. Variables y Tipos de Datos
Las variables son "cajas" donde guardamos información. El tipo de dato define qué forma tiene esa información para que la computadora sepa cómo procesarla.
* **Enteros (`int`):** Números sin decimales (Ejemplo: `10`, `-5`).
* **Flotantes (`float`):** Números con decimales (Ejemplo: `3.14`, `-0.5`).
* **Cadenas de texto (`string` / `str`):** Palabras o caracteres encerrados en comillas (Ejemplo: `"Hola"`, `"123"`).
* **Booleanos (`bool`):** Valores de verdad, solo pueden ser Verdadero o Falso (`True`, `False`).

### 3. Operadores
Son símbolos que le indican a la computadora qué operación matemática o lógica realizar.
* **Aritméticos:** Realizan cálculos matemáticos. Suma (`+`), Resta (`-`), Multiplicación (`*`), División (`/`), División entera (`//`, devuelve el resultado sin decimales), Módulo (`%`, devuelve el residuo de una división).
* **Comparativos:** Comparan dos valores y devuelven un Booleano (`True` o `False`). Igualdad (`==`), Mayor que (`>`), Menor que (`<`), Mayor o igual (`>=`), Menor o igual (`<=`), Diferente (`!=`).
* **Lógicos:** Permiten combinar múltiples condiciones. `and` (Y: ambas deben ser verdaderas), `or` (O: al menos una debe ser verdadera), `not` (NO: invierte el valor de verdad).

### 4. Estructuras de Control (Condicionales)
Permiten que el programa tome decisiones basándose en condiciones, como si fuera un cruce de caminos.
* **`if` / `elif` / `else`:** Ejecuta un bloque de código SI una condición es verdadera. Si no lo es, puede evaluar otra condición (`elif`), o ejecutar una acción por defecto (`else`).
* **`match` / `case`:** Funciona como un "interruptor múltiple". Es ideal cuando una sola variable puede tener muchos valores exactos posibles y queremos hacer algo diferente para cada uno.

### 5. Ciclos (Bucles)
Permiten repetir un bloque de código varias veces, ahorrando tiempo y líneas de programación.
* **`while` (Mientras):** Repite el código *mientras* una condición siga siendo verdadera. Es útil cuando no sabemos cuántas veces se debe repetir algo.
* **`for` (Para):** Repite el código un número específico de veces o recorre los elementos de una colección (como una lista).
* **`break`:** Actúa como un freno de emergencia. Termina el ciclo inmediatamente, sin importar si la condición se seguía cumpliendo.
* **`continue`:** Salta el resto del código en la iteración actual y pasa directamente a la siguiente repetición del ciclo.


### 6. Estructuras de Datos Simples
* **Listas:** Son colecciones ordenadas que pueden guardar múltiples datos (incluso de diferentes tipos). Se pueden modificar. Se crean con corchetes `[]`.
* **Tuplas:** Similares a las listas, pero son **inmutables**; una vez creadas, no se pueden modificar, agregar ni eliminar elementos. Se crean con paréntesis `()`.

### 7. Manipulación de Listas
* **`len(lista)`:** Devuelve la cantidad total de elementos que contiene la lista (o los caracteres de un texto).
* **Indexación negativa:** `lista[-1]` accede al último elemento, `[-2]` al penúltimo, etc.
* **`append(valor)`:** Agrega un nuevo elemento al final de la lista.
* **`pop()`:** Elimina y devuelve el último elemento de la lista (o el índice que se le indique).
* **`remove(valor)`:** Busca un valor específico y elimina su primera aparición.
* **`del lista[índice]`:** Elimina un elemento directamente por su posición, sin devolver su valor.

### 8. Funciones
Son bloques de código reutilizables diseñados para realizar una tarea específica. Piensen en ellas como "mini-programas" dentro de su programa.
* **Declaración vs Invocación:** *Declarar* (`def`) es construir la máquina y enseñarle cómo funciona. *Invocar* es presionar el botón de encendido para que la máquina trabaje.
* **Parámetros:** Son las ranuras por donde introducimos la materia prima (datos) que la función necesita para trabajar.
* **`return`:** Es el producto final. Es la forma en que la función devuelve el resultado de su trabajo al programa principal para seguir siendo utilizado.


## Parte 2: Listas de Listas (Matrices)

Hasta ahora, hemos visto listas que parecen una línea recta de cajas. Pero, ¿qué pasa si necesitamos organizar la información en una tabla o cuadrícula? Aquí entran las **Listas de Listas**, también conocidas como **Matrices**.

**Analogía:** Imaginen una hoja de cálculo de Excel o un edificio de apartamentos. El edificio tiene varios pisos (filas). Si ustedes van a un piso específico, encontrarán varios apartamentos (columnas o elementos individuales en esa fila).



### Ejemplo Conceptual: Inventario de Supermercado

| Producto | Precio | Cantidad |
|----------|--------|----------|
| Arroz    | 1200   | 5        |
| Leche    | 900    | 10       |
| Pan      | 700    | 8        |

### Representación en Python

Para traducir esta tabla a Python, creamos una lista gigante que actúa como el "edificio". Dentro de ella, cada fila de la tabla será una lista más pequeña.

```python
# Sintaxis: Se abren corchetes generales, y adentro se separan por comas otros corchetes.
inventario = [
    ["Arroz", 1200, 5],  # Fila 0
    ["Leche", 900, 10],  # Fila 1
    ["Pan", 700, 8]      # Fila 2
]
```

### 2. ¿Cómo acceder a los datos? (Doble Índice)

Para encontrar a una persona en un edificio, necesitan dos datos: el piso y el número de puerta. En las matrices de Python es igual, usamos un **Doble Índice**: `matriz[fila][columna]`.

Recuerden que en Python **siempre empezamos a contar desde cero**.

```python
print("--- SISTEMA DE BÚSQUEDA POR COORDENADAS ---")

bodega = [
    ["Arroz", "Frijoles", "Lentejas"], # Fila 0: Granos
    ["Atún", "Sardina", "Sopa"],       # Fila 1: Enlatados
    ["Agua", "Jugo", "Refresco"]       # Fila 2: Bebidas
]

# OBJETIVO 1: Extraer la "Sardina".
# Paso 1: ¿En qué fila está? En la fila 1 (Enlatados).
# Paso 2: ¿En qué columna está dentro de esa fila? En la columna 1.
producto_buscado = bodega[1][1]
print("Producto extraído:", producto_buscado) # Imprime: Sardina

# OBJETIVO 2: Extraer el "Agua". 
# Está en la Fila 2, Columna 0.
print("Bebida seleccionada:", bodega[2][0]) # Imprime: Agua

# OBJETIVO 3: Modificar un dato específico.
# El "Jugo" (Fila 2, Columna 1) se agotó. Lo reemplazaremos por "Té".
bodega[2][1] = "Té"
print("Fila de bebidas actualizada:", bodega[2]) 
# Imprime: ['Agua', 'Té', 'Refresco']
```

# Recorrer una matriz

Si queremos revisar absolutamente todos los productos de la bodega, necesitamos un **bucle anidado** (un ciclo `for` dentro de otro ciclo `for`).

**Analogía del Reloj:** Piensen en las manecillas de un reloj. El ciclo interno (minutero) tiene que dar una vuelta completa (revisar todos los elementos de un piso) antes de que el ciclo externo (horario) avance un solo paso (baje al siguiente piso).

### Forma 1: Recorrido de Lectura (Para ver los datos)

Usamos este método cuando solo queremos "leer" o imprimir los datos, pero no necesitamos cambiar su posición en la estructura.

```python
matriz_principal = [
    ["Manzanas", "Peras"],
    ["Leche", "Queso"]
]

# El primer bucle se encarga de bajar por los pisos (filas)
for fila in matriz_principal:
    # El segundo bucle camina de izquierda a derecha por los productos del piso actual
    for elemento in fila:
        print("Revisando producto:", elemento)
```

# Reto 

### Reto Práctico - El Estante del Supermercado

**El Problema:** Tenemos un estante representado como una matriz. Nuestro trabajo es programar un robot que recorra todo el estante. Si encuentra el producto "agua", debe reemplazarlo por la palabra "vacio" (porque alguien lo compró) y luego detener su búsqueda en esa fila.

A continuación, veremos dos formas de resolverlo. Es crucial entender por qué la segunda forma es la más segura y recomendada por los programadores profesionales.

### Solución y Análisis de Código

```python
# Definimos nuestro estante inicial
estante = [
    ["manzana", "pera", "uva"],
    ["leche", "agua", "queso"],
    ["jabon", "cloro", "agua"]
]

def mostrar_productos(matriz):
    """Función de apoyo para imprimir el estante piso por piso"""
    print("-" * 20)
    for fila in matriz:
        print(fila)
    print("-" * 20)

print("Estante original:")
mostrar_productos(estante)

# =====================================================================
# MÉTODO 1: Usando variables temporales e index()
# Advertencia: Es funcional para cosas simples, pero peligroso si hay
# elementos duplicados, ya que index() siempre modifica el primero que halla.
# =====================================================================

# Hacemos una copia para no dañar el original en el ejemplo
estante_metodo1 = [fila.copy() for fila in estante]

# Ciclo externo: viaja por las filas completas (pisos)
for fila in estante_metodo1:
    # Ciclo interno: viaja por cada producto dentro de la fila actual 
    for producto in fila:
        if producto == "agua":
            # Buscamos la posición del producto en la fila actual y lo cambiamos
            posicion = fila.index(producto)
            fila[posicion] = "vacio"
            break # Rompe el ciclo interno, pasa a la siguiente fila

print("Resultado del Método 1 (Lectura directa):")
mostrar_productos(estante_metodo1)


# =====================================================================
# MÉTODO 2: Usando índices con range(len()) - ¡EL MÉTODO RECOMENDADO!
# Al trabajar directamente con coordenadas matemáticas, tenemos control
# absoluto y no importa si hay elementos repetidos.
# =====================================================================

estante_metodo2 = [fila.copy() for fila in estante]

# len(estante) nos dice cuántas filas hay. En este caso 3.
# range(3) genera los números: 0, 1, 2.
for indice_fila in range(len(estante_metodo2)):
    
    # len(estante[indice_fila]) nos dice cuántos elementos hay en la fila actual.
    # range(3) genera los números de las columnas: 0, 1, 2.
    for indice_columna in range(len(estante_metodo2[indice_fila])):
        
        # Leemos el dato usando sus coordenadas exactas
        if estante_metodo2[indice_fila][indice_columna] == "agua":
            
            # Como tenemos las coordenadas, lo reemplazamos directamente por "pepsi"
            estante_metodo2[indice_fila][indice_columna] = "pepsi"
            break # Terminamos de buscar en esta fila

print("Resultado del Método 2 (Por coordenadas):")
mostrar_productos(estante_metodo2)
```

**Nota para los estudiantes:** Observen cómo en el Método 2 usamos `range(len())`. Al programar, siempre que necesiten **modificar** una estructura de datos mientras la recorren, hacerlo a través de sus índices o coordenadas evitará comportamientos extraños y errores de lógica.
