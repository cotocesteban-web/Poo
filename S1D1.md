# Guía de Estudio: Introducción a la Programación Orientada a Objetos (POO)

## El Cambio de Paradigma

**¿Qué es un Paradigma?**
Un paradigma no es un nuevo lenguaje de programación, es simplemente una nueva forma de organizar nuestras ideas y estructurar las soluciones. 

**La Analogía de la Bodega:**
Imagina que en nuestros programas anteriores (Programación Estructurada), éramos gerentes que anotaban los nombres de los productos en una lista de papel, los precios en otra lista y las cantidades en otra. Si se nos perdía una hoja o se desordenaba una lista, el sistema colapsaba (el "Pan" terminaba costando lo mismo que un "Televisor"). 

Con la **Programación Orientada a Objetos (POO)**, dejamos de usar listas sueltas. En su lugar, construimos "cajas fuertes" inteligentes llamadas **Objetos**. Cada Objeto (por ejemplo, el producto "Leche") guarda dentro de sí mismo su propio nombre, su propio precio y su propia cantidad, y nadie desde afuera puede desordenarlo.

## 1. Convenciones de Nombres: El Idioma de los Ingenieros

Cuando trabajamos en equipos reales, otro programador debe saber exactamente qué es cada palabra en tu código con solo mirarla. Para esto, existen reglas ortográficas estrictas.

### Regla de Oro 1: PascalCase (Exclusivo para Clases)
* **Definición:** Es el formato donde cada palabra inicia con letra Mayúscula. No se permiten espacios y **está estrictamente prohibido usar guiones bajos**.
* **Uso:** Se utiliza única y exclusivamente para nombrar **Clases** (nuestros planos arquitectónicos).
* **Ejemplos Correctos:** `Cliente`, `ClienteFrecuente`, `VehiculoDeReparto`.
* **Ejemplos Incorrectos:** `cliente`, `Cliente_Frecuente`, `vehiculoReparto`.

### Regla de Oro 2: snake_case (Para Funciones y Variables)
* **Definición:** Es el formato donde todas las letras van en minúscula. Si el nombre tiene varias palabras, se unen simulando una serpiente en el suelo usando un guión bajo (`_`).
* **Uso:** Se utiliza para todo lo demás (variables, funciones, listas, diccionarios).
* **Ejemplos Correctos:** `calcular_pago()`, `cliente_frecuente`, `vehiculo_asignado`.
* **Ejemplos Incorrectos:** `CalcularPago()`, `ClienteFrecuente`, `vehiculoAsignado`.

### Código de Ejemplo: Contraste Visual

    pass

# PASO 2: Variables y Funciones (El uso diario)

## 2. Fabricando la Realidad: Clases y Objetos

### ¿Qué es una Clase?
Una Clase es un concepto abstracto. Es el **plano arquitectónico** de una casa o el **molde de metal** para hacer galletas. Le enseña a la computadora qué forma tendrá algo, pero no existe físicamente; no puedes vivir en un plano de papel ni te puedes comer un molde de metal.

### ¿Qué es un Objeto (Instancia)?
Un Objeto es la manifestación física y real de ese plano. Es la casa ya construida en la que puedes entrar, o la galleta horneada que te puedes comer. 
A la acción de presionar el molde contra la masa para crear un objeto real se le llama **Instanciación**. A partir de un solo molde, puedes fabricar miles de objetos independientes.

```python
# 1. Creamos el molde de metal (La Clase abstracta)
class ClienteFrecuente:
    pass

# 2. La Instanciación (Presionamos el molde sobre la masa)
# Creamos un objeto real y lo guardamos en la variable cliente1
cliente1 = ClienteFrecuente()

# Presionamos el molde otra vez para crear un SEGUNDO objeto independiente
cliente2 = ClienteFrecuente()

# Si imprimimos el molde, la computadora dice que es solo una definición:
print("El Molde:", ClienteFrecuente)

# Si imprimimos los objetos, la computadora nos muestra que son entes físicos en la memoria:
print("Objeto 1:", cliente1)
print("Objeto 2:", cliente2)
```

## 3. Comprobando el ADN del Objeto: type() e id()

Cuando tenemos muchos objetos, necesitamos herramientas para examinarlos:
* **`type()` (La prueba de ADN):** Nos dice exactamente qué molde (Clase) se utilizó para fabricar ese objeto.
* **`id()` (La Cédula de Identidad):** Nos devuelve la dirección exacta en la memoria RAM donde vive el objeto. Esto demuestra que, aunque sean gemelos nacidos del mismo molde, son individuos separados con su propia "cédula".

```python
class Factura:
    pass

factura_enero = Factura()
factura_febrero = Factura()

# Comprobando de qué molde vienen:
print("Molde de origen:", type(factura_enero))

# Comprobando que son entidades únicas en la memoria:
print("Cédula de la factura de enero:", id(factura_enero))
print("Cédula de la factura de febrero:", id(factura_febrero))
```

### Ejercicio 3: Gestión de Flota, Listas y Control de Capacidad

Imaginen que son los dueños de una empresa de transportes. Tienen un plano llamado `Camion`. Aunque el plano es solo un papel con la palabra `pass` (un plano en blanco), cada vez que mandan a fabricar un camión con ese plano, la fábrica les entrega una unidad física que ocupa un espacio en su estacionamiento. 

### Instrucciones del Ejercicio

1.  **Definición de Moldes:** Crea dos clases vacías llamadas `Camion` y `CentroLogistico` usando la palabra clave `pass`.
2.  **Fabricación en Serie:** Fabrica cinco (5) objetos a partir de la clase `Camion`. Guárdalos en variables llamadas `camion1`, `camion2`, `camion3`, `camion4` y `camion5`.
3.  **El Estacionamiento (Listas):** Crea una lista llamada `garaje_principal` y guarda dentro de ella los 5 objetos de camión que acabas de fabricar.
4.  **Conteo y Aritmética:** * Calcula el total de camiones usando la función `len()` sobre la lista. 
    * Imagina que cada camión paga un impuesto de circulación de **500 dólares**. Crea una variable llamada `impuesto_total` que multiplique la cantidad de camiones por 500.
5.  **Revisión de Identidad (Ciclos):** Usa un ciclo `for` para recorrer la lista `garaje_principal`. Por cada camión en la lista, imprime su dirección de memoria usando la función `id()`. Esto demuestra que, aunque todos vienen del mismo molde, cada uno es único.
6.  **Control de Capacidad (Condicionales):** Supongamos que el centro logístico solo tiene capacidad para **4 camiones**. 
    * Escribe una estructura `if` que compare si la cantidad de camiones en la lista es mayor a 4. 
    * Si es mayor, imprime un mensaje de alerta: "¡Capacidad excedida! Debes mover camiones a otra sucursal".
    * Si no, imprime: "Capacidad óptima".

```python
print("\n--- EJERCICIO 3: LOGÍSTICA Y CONTROL ---")

# 1. Definición de moldes
class Camion:
    pass

class CentroLogistico:
    pass

# 2. Fabricación en serie de 5 objetos Camion
camion1 = Camion()
camion2 = Camion()
camion3 = Camion()
camion4 = Camion()
camion5 = Camion()

# 3. El Estacionamiento: Creamos una lista y guardamos los objetos adentro
garaje_principal = [camion1, camion2, camion3, camion4, camion5]

# 4. Conteo y Aritmética
# Usamos len() para contar cuántos objetos hay dentro de la lista
total_camiones = len(garaje_principal)

# Calculamos los impuestos operativos
impuesto_por_camion = 500
impuesto_total = total_camiones * impuesto_por_camion
print(f"Total a pagar en impuestos por {total_camiones} camiones: ${impuesto_total}")

print("\nRevisión de identidades en el estacionamiento:")
# 5. Revisión de Identidad (Ciclos)
# Recorremos la lista. En cada vuelta, 'vehiculo' representa un objeto distinto
for vehiculo in garaje_principal:
    print(f"Inspeccionando camión con ID de memoria: {id(vehiculo)}")

print("\n--- Control de Capacidad ---")
# 6. Control de Capacidad (Condicionales)
# Verificamos si la cantidad de objetos supera el límite físico del edificio
capacidad_maxima = 4

if total_camiones > capacidad_maxima:
    print("¡ALERTA ROJA! Capacidad excedida.")
    print(f"Tenemos {total_camiones} camiones pero solo caben {capacidad_maxima}.")
    print("Debes mover camiones a otra sucursal inmediatamente.")
else:
    print("Capacidad óptima. Todo bajo control.")
```
